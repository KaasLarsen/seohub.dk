SEO tip #9: FAQ Schema (JSON-LD) – hurtige rich results (2025)

Få mere SERP-plads og højere CTR ved at tilføje gennemarbejdet FAQ Schema til dine vigtigste sider.

TL;DR

  • Vælg 3–6 ægte FAQ-spørgsmål, der matcher brugerens søgeintention.
  • Implementér JSON-LD (ikke microdata) – placer i <head> eller nederst i <body>.
  • Undgå duplicering på tværs af sider, og sørg for, at Q/A faktisk findes i synligt indhold.
  • Test med Rich Results Test, og overvåg i Search Console.

Hvorfor FAQ Schema?

FAQ-udvidelser kan give ekstra synlighed direkte i SERP’en, hvilket ofte øger CTR. De hjælper også Google med at forstå din side, og de fungerer som “hurtige svar” der matcher brugerens mikro-intentioner.

Sådan implementerer du FAQ Schema korrekt

  1. Udvælg 3–6 spørgsmål, der reelt bliver stillet – brug “Folk spørger også” og din egen support/indsigt.
  2. Besvar kort og præcist (2–5 linjer). Én idé pr. svar.
  3. Byg JSON-LD – brug gerne vores FAQ-schema generator.
  4. Indsæt scriptet på siden (og sørg for, at spørgsmål/svar også vises i selve indholdet).
  5. Test i Rich Results Test, og hold øje med rapporter i Search Console.

Mini-eksempel på JSON-LD

{
  "@context": "https://schema.org",
  "@type": "FAQPage",
  "mainEntity": [
    {
      "@type": "Question",
      "name": "Hvad er FAQ Schema?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Struktureret data, der markerer spørgsmål/svar, så Google kan vise uddrag i SERP’en."
      }
    },
    {
      "@type": "Question",
      "name": "Hvor mange spørgsmål bør jeg tilføje?",
      "acceptedAnswer": {
        "@type": "Answer",
        "text": "Typisk 3–6. Fokusér på kvalitet og relevans frem for kvantitet."
      }
    }
  ]
}

Generér hele din FAQ hurtigere i vores FAQ-generator, og tilpas frit.

Faldgruber du bør undgå

  • FAQ-indhold findes ikke på siden (Google vil ofte ignorere markup).
  • Overdreven brug på alle sider (brug hvor det giver mening).
  • Salgsagtige eller manipulerende svar – hold det sagligt.
  • Duplikerede Q/A på mange URL’er.
