Google Search Console guide (2025)

Den praktiske guide til opsætning, rapporter og fejlretning – så du kan udnytte GSC til vækst i 2025.

TL;DR

  • Brug Domain property (DNS) så alle subdomæner/protokoller dækkes.
  • Indsend sitemap, gennemgå Page indexing og fix udelukkelser ved kilden.
  • Find vækst i Performance via lav CTR, side‑2‑kandidater og long‑tail spørgsmål.
  • Prioritér Core Web Vitals på de sider/skabeloner der har mest trafik.
  • URL Inspection til nye/ændrede sider → anmod om indeksering efter rettelser.

Google Search Console (GSC) giver dig indsigt i, hvordan Google crawler, indekserer og viser dit website i søgeresultaterne. I 2025 handler det om kvalitet i indeks, stærk UX (CWV) og høj relevans. Her får du en trin-for-trin guide samt konkrete tjeklister og arbejdsgange, du kan bruge månedligt.

Kom i gang: opsætning & verifikation

  1. Opret Domain property i GSC og tilføj DNS TXT-record hos din udbyder.
  2. Alternativt: URL‑prefix hvis du ikke kan ændre DNS (kræver varianter www/https).
  3. Tilføj brugere under Settings → Users and permissions (least privilege).
  4. Associations: Link til GA4 for krydsdata.
  5. Bekræft at den rigtige version (https + www/uden www) er kanonisk i dit CMS.

Sitemaps, crawling & indexering

  • Indsend sitemap i Index → Sitemaps (fx /sitemap.xml med undersitemaps).
  • Robots.txt: Tjek at vigtige sektioner ikke er blokeret. Brug noindex korrekt.
  • Page indexing: Find årsager til udelukkelser (noindex, redirect, canonical, soft 404, 404, blocked).
  • Fix ved kilden (CMS/tema). URL Inspection → Request indexing efter rettelse.

Experience & Core Web Vitals (CWV)

Brug rapporten til at prioritere LCP, INP og CLS der rammer sider med mest trafik. Optimer billeder (format/størrelse), lazyload, kritisk CSS og reducer JS‑blokeringer.

  • Skabeloner først: Løs problemer i tema/komponenter så forbedringer skalerer.
  • Mål igen: Udrul → vent CrUX opdatering → bekræft i GSC.

URL Inspection: debugging af enkeltsider

  • Nye sider: “Request indexing”.
  • Opdaterede sider: Anmod om genindeksering ved større ændringer.
  • Canonical‑tjek: Sammenlign “User‑declared” vs. “Google‑selected”.

Google Discover: sådan øger du chancen

  • E‑E‑A‑T: Tydelig forfatter/brand, kilder og opdaterede fakta.
  • Visuelt stærkt: Store, skarpe billeder (og korrekte Open Graph‑tags).
  • Aktualitet: Trends/ny vinkel på evergreen‑emner.

Links, sikkerhed & manuelle tiltag

  • Links: Se henvisende domæner og top‑linked pages → styrk interne links til målsider.
  • Manual actions: Læs årsag, udbedr og indsende reconsideration.
  • Security issues: Malware/hacked content → ret straks og anmod om gennemgang.

Anbefalet arbejdsgang (månedlig)

  1. Tjek Performance og lav CTR‑forbedringer + find side‑2‑kandidater.
  2. Gennemgå Page indexing og fix kritiske udelukkelser først.
  3. Se Experience/CWV på top‑sider og planlæg optimeringer.
  4. Kig Links (nye stærke) og forstærk interne links.
  5. Overvåg Manual/Security og Removals.
