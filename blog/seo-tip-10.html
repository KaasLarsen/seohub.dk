SEO tip #10: Sitelinks Search Box – sådan får du søgeboks i Google

Brug WebSite + SearchAction JSON-LD korrekt, så Google kan vise intern søgning direkte i dit brand-resultat.

TL;DR

  • Tilføj WebSite JSON-LD med SearchAction (også for “target” + “query-input”).
  • Din interne søgning skal virke offentligt (ingen login), og returnere relevante resultater.
  • Brug et stabilt søge-URL-mønster (fx /soeg?q= eller /?s=).
  • Google vælger selv at vise boksen – korrekt markup + god brandtrafik øger chancen.

Hvorfor en Sitelinks Search Box?

En søgeboks under dit brand-resultat lader brugerne hoppe direkte til interne søgeresultater på dit site. Det kan reducere pogo-sticking, give hurtigere svar og øge engagement på vigtige sider.

Implementering trin for trin

  1. Sørg for, at din interne søgning er tilgængelig for alle (ingen login, ingen blokering i robots.txt).
  2. Brug et konsistent søge-parameter (fx https://eksempel.dk/soeg?q=sporgsmaal).
  3. Tilføj WebSite JSON-LD med SearchAction (se skabelon nedenfor).
  4. Test i Rich Results Test og overvåg Search Console.

Skabelon: WebSite + SearchAction (JSON-LD)

{
  "@context": "https://schema.org",
  "@type": "WebSite",
  "url": "https://www.eksempel.dk/",
  "potentialAction": {
    "@type": "SearchAction",
    "target": "https://www.eksempel.dk/soeg?q={search_term_string}",
    "query-input": "required name=search_term_string"
  }
}

Tip: Har din søgning et andet parameter (fx ?s=), så ret target til https://www.eksempel.dk/?s={search_term_string}.

UX, kvalitet og måling

  • Vis tydelige titler, uddrag og brødkrummer i interne søgeresultater.
  • Sortér efter relevans; håndtér stavefejl og synonymer, hvis muligt.
  • Mål på sessions fra brand-søgninger, samt klikdybde og konverteringer.
