Core Web Vitals i 2025: sådan forbedrer du LCP, INP og CLS

Core Web Vitals er Googles vigtigste hastigheds- og oplevelsesfaktorer. Her får du en opdateret 2025-guide med konkrete forbedringer.

Opdateret: 14. august 2025 • Læsetid: 6–8 min

Hvad er Core Web Vitals i 2025?

Core Web Vitals består i år af tre målepunkter: LCP (Largest Contentful Paint), INP (Interaction to Next Paint) og CLS (Cumulative Layout Shift). De måler henholdsvis hastighed, interaktivitet og visuel stabilitet.

LCP – Largest Contentful Paint

Mål: LCP under 2,5 sekunder.

  • Optimer store billeder (komprimering, moderne formater som WebP/AVIF).
  • Lazy-load billeder under folden.
  • Brug hurtigere hosting eller CDN til tunge ressourcer.

INP – Interaction to Next Paint

Mål: INP under 200 ms.

  • Reducer tung JavaScript – split op og indlæs kun nødvendigt JS.
  • Fjern unødvendige 3. parts scripts (tracking, widgets).
  • Brug requestIdleCallback eller async loading til ikke-kritiske scripts.

CLS – Cumulative Layout Shift

Mål: CLS under 0,1.

  • Angiv faste størrelser for billeder og video-elementer i HTML/CSS.
  • Undgå dynamiske elementer, der skubber indhold ned, uden foruddefineret plads.
  • Indlæs skrifttyper korrekt for at undgå layout hop.

Sådan tester du dine Core Web Vitals

  • Google PageSpeed Insights (PSI).
  • Chrome Lighthouse.
  • Google Search Console → Core Web Vitals rapport.

Quick wins

  • Konverter billeder til WebP.
  • Brug preload på de vigtigste skrifttyper.
  • Fjern eller udskyd tunge scripts.
  • Brug en moderne caching-strategi (service workers, edge caching).
