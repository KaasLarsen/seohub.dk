Core Web Vitals i 2025 – sådan scorer du grønt i PageSpeed Insights

Praktiske forbedringer af LCP, INP og CLS – fra billedeoptimering og font-loading til JS-split, caching og hurtig hosting.

TL;DR

  • LCP < 2,5s: Komprimer hero-billeder, brug preload og moderne formater (WebP/AVIF).
  • INP: Minimer JS, split koden, udsæt 3. parts scripts og brug passiv event listeners.
  • CLS < 0,1: Reservér plads til billeder/ifram es (width/height), font-display: swap, undgå layout-jumps.
  • Serverhastighed: Hurtig hosting og caching CDN → bedre scores.
  • Mål løbende i PageSpeed Insights og Search Console (CWV-rapporten).

Hvad er Core Web Vitals?

Core Web Vitals er Googles sæt af brugercentrerede metrikker, der måler oplevet hastighed og stabilitet:

  • LCP (Largest Contentful Paint): hvor hurtigt det største synlige element loader.
  • INP (Interaction to Next Paint): hvor hurtigt siden reagerer på interaktioner.
  • CLS (Cumulative Layout Shift): hvor stabilt layoutet er under indlæsning.

1) LCP – gør det største element hurtigt

  • Komprimer hero-billeder (WebP/AVIF) og brug ret dimension.
  • Preload det vigtigste billede: <link rel="preload" as="image" href="/path.webp" imagesrcset="..." imagesizes="...">
  • Reducer CSS-blokering: kritisk CSS inline, resten udsættes.
  • Brug hurtig hosting og CDN.

2) INP – responsiv interaktion

  • Split store JS-bundles; indlæs kun det nødvendige pr. side.
  • Udsæt tunge 3. parts scripts, og brug defer.
  • Undgå lange main-thread tasks (>50 ms).
  • Brug passiv event listeners til scroll/touch.

3) CLS – stabilt layout

  • Angiv width/height på billeder, og reserver plads til embeds/iframes.
  • Brug font-display: swap og preconn ect til font-domæner.
  • Indsæt ikke DOM-elementer over eksisterende indhold, når siden er interaktiv.

4) Quick wins du kan lave i dag

  • Komprimer alle billeder > 200 KB → WebP/AVIF og lazyload.
  • Fjern ubrugte JS/CSS, og defer alt ikke-kritisk JS.
  • Cache statiske assets med lang max-age + immutable.
  • Brug systemfonte eller hostede fon te med swap.
  • Reducer antallet af 3. parts scripts.

5) Hosting & CDN

Serverhastighed har stor effekt på både LCP og INP. Vælg en hurtig hostingløsning, gerne med HTTP/2/3, SSD og god TTFB.
