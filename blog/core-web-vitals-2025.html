Core Web Vitals i 2025: sådan forbedrer du LCP, INP og CLS

Opdateret 13. august 2025 • Læsetid: –

Core Web Vitals (CWV) måler brugeroplevelse i den virkelige verden. I 2025 er fokus især på LCP (Largest Contentful Paint), INP (Interaction to Next Paint) og CLS (Cumulative Layout Shift). Her får du konkrete forbedringer, hvordan du måler, og en tjekliste du kan følge trin for trin.

Hurtig oversigt over gode grænser

  • LCP: ≤ 2,5 s
  • INP: ≤ 200 ms
  • CLS: ≤ 0,1

LCP – gør det primære indhold hurtigt synligt

LCP rammes typisk af tunge hero-billeder, langsomme servere eller blokkerende ressourcer.

Quick wins

  • Konverter hero-billeder til WebP/AVIF og servér korrekte størrelser (srcset/sizes).
  • Preload den største hero-ressource: 
    <link rel="preload" as="image" href="/img/hero@2x.avif" imagesrcset="/img/hero.avif 1x, /img/hero@2x.avif 2x" imagesizes="100vw">
  • Flyt tunge 3.-parts scripts ned eller brug defer/async.
  • Aktivér HTTP/2 eller HTTP/3 og cache-control for statiske assets.

Diagnose

  • Brug PageSpeed Insights → “Opportunities” for at finde LCP-blokkere.
  • WebPageTest → “Filmstrip” for at se hvad der vises hvornår.

INP – gør interaktioner hurtige og responsive

INP måler hvor hurtigt siden reagerer på alle interaktioner (klik, tast, tap). Problemer skyldes typisk tung JavaScript.

Quick wins

  • Code-splitting: indlæs kun det JS, der er nødvendigt på den aktuelle side.
  • Debounce dyre event-handlers (søgning, input) og brug passive event listeners: 
    window.addEventListener('touchstart', onTouch, { passive: true });
  • Offload tunge beregninger til Web Workers.
  • Fjern/udskift tunge UI-libs (store frameworks for simple sider).

Diagnose

  • Chrome Performance profiler → se “Main thread” blokeringer.
  • INP i CrUX/Field Data → prioriter de sider/flows med dårligst tal.

CLS – stop layout-hop

CLS opstår når indhold flytter sig efter render (annoncer, billeder uden dimensioner, webfonts).

Quick wins

  • Angiv altid width/height på billeder eller brug moderne CSS med aspect-ratio.
  • Reserver plads til annoncer/embeds med en container i fast højde.
  • Håndtér webfonts: font-display: swap + <link rel="preload" as="font">.

Kodeeksempel: reserve plads til billede

<style>
.hero { aspect-ratio: 16/9; background:#eee; }
</style>
<img class="hero" src="/img/hero.avif" alt="Hero">

Cache & levering af assets

  • Brug lange cache-tider for fingerprintede filer (immutable), kortere for HTML.
  • Komprimer med Brotli; sørg for GZIP fallback.
  • Preconnect til vigtige domæner: 
    <link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>

Måling: lab vs. field data

Lab (syntetisk) = stabile forhold, gode til at finde problemer. Field (rigtige brugere) = sandheden i CrUX/GA.

  • Start i PageSpeed (lab) → implementér fixes → verificér i CrUX (field).
  • Hold øje med “p95” i field data (de langsomste 5%).

Tjekliste (printbar)

  • [ ] LCP ≤ 2,5 s: WebP/AVIF + preload af hero, fjern render-blockers
  • [ ] INP ≤ 200 ms: split JS, debounce events, workers for tunge tasks
  • [ ] CLS ≤ 0,1: faste dimensioner, reserverede containere, font-display
  • [ ] Cache: immutable på assets, kort for HTML
  • [ ] HTTP/2 eller HTTP/3, Brotli aktiv
  • [ ] Verificér forbedringer i CrUX/GA
