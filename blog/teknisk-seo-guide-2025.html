Komplet teknisk SEO-guide (2025)

Alt om crawling, Core Web Vitals, mobile-first, HTTPS, canonical, redirects, hreflang og structured data – forklaret trin-for-trin med eksempler og tjekliste.

TL;DR

  • Sitemap + robots.txt skal hjælpe (ikke hindre) crawling & indexering.
  • Core Web Vitals (LCP, CLS, INP) påvirker både UX og synlighed – mål og forbedr løbende.
  • HTTPS, canonical & 301-redirects fjerner duplikerede versioner og sikrer korrekt signal.
  • Mobile-first: Optimer primært på mobil; test alt på små skærme først.
  • Structured data (JSON-LD) gør indholdet tydeligt for Google og kan udløse rich results.

Teknisk SEO er fundamentet, der gør det muligt for dine tekster og links at virke. Uden korrekt crawling, hastighed og struktur får du sjældent det fulde udbytte. Denne guide giver dig en praktisk step-by-step tilgang – med små tjeklister og eksempler, som du kan implementere med det samme.

Tip: Brug vores gratis værktøjer undervejs: SERP & Meta, Robots.txt generator, Sitemap.xml generator og Intern linkbuilder.

Crawling & indexering

Robots.txt

Brug robots.txt til at styre crawl-adgang, men blokér ikke sider, du ønsker i indekset.

Vis eksempel (minimal robots.txt) 
User-agent: *
Disallow:
Sitemap: https://www.seohub.dk/sitemap.xml

XML-sitemap

Et opdateret sitemap hjælper Google med at finde nyt og opdateret indhold. Sørg for at alle vigtige skabeloner er med: blog, værktøjer, ordbog m.m.

Crawl budget

Store sites bør fjerne støj (tynde sider, filtre, dubletter), så crawl-budget bruges på det, der betyder noget. Læs mere i ordbogen: Crawl budget.

Tjekliste: Crawling
  • Robots.txt blokerer ikke vigtige skabeloner.
  • Sitemap indeholder alle centrale URL’er (ingen 404/410/301).
  • Parametre håndteres (noindex, canonical eller blokering efter behov).

Site speed & Core Web Vitals

Fokuser på LCP (load-hastighed), CLS (stabilitet) og INP (interaktivitet). Start med billeder, fonte og tredjeparts-scripts.

Hurtige wins
  • Komprimer billeder (WebP/AVIF) og brug width/height + loading="lazy".
  • Preload vigtigste font og hero-billede.
  • Udsæt ikke-kritiske scripts (defer) og fjern ubrugte.

Se ordbogen: Core Web Vitals, CLS.

Mobile-first & UX

Design på mobil først. Test navigation, tap-mål, tabeller og interaktioner på små skærme.

  • Responsivt grid, læsbare fonte, nok kontrast.
  • Undgå intrusive popups – især på mobil.
  • Hold CTA’er synlige, men ikke aggressive.

HTTPS, canonical & redirects

Brug HTTPS overalt. Canonical for at pege på den “rigtige” version, og 301-redirects til at samle signaler, når URL’er ændrer sig.

Eksempel: 301 i .htaccess 
RewriteEngine On
RewriteCond %{HTTPS} off
RewriteRule ^(.*)$ https://www.seohub.dk/$1 [L,R=301]

Se ordbogen: 301 redirect, Canonical tag.

Duplicate content & index bloat

Filtre, søgning, tags og arkiver skaber ofte uønskede dubletter. Ryd op med noindex, canonical og logisk IA.

  • Noindex på paginerede/tynde lister, der ikke skal rangere.
  • Canonical mellem varianter (sprog, parametre, UTM’er).
  • Fjern døde sider (410) og ryd op i interne links.

International SEO (hreflang)

Hreflang hjælper Google med at vise den rette sprog-/landeversion. Vær konsekvent i sprogkoder og URL-struktur.

Structured data (schema)

Brug JSON-LD til at beskrive dine sider (Article, Product, FAQ). Det hjælper Google med at forstå indholdet.

Eksempel: Article (JSON-LD) 
{
  "@context": "https://schema.org",
  "@type": "Article",
  "headline": "Min artikel",
  "author": {"@type":"Person","name":"Navn"},
  "datePublished": "2025-08-26"
}

Tjekliste: Teknisk SEO (2025)

  • HTTPS og HSTS aktiveret, ingen mixed content.
  • Kun én version af domænet er tilgængelig (www/non-www, http→https).
  • XML-sitemap opdateres automatisk og kun med 200-URL’er.
  • Robots.txt blokkerer ikke vigtige skabeloner; sitemap erklæret.
  • Core Web Vitals: LCP < 2.5s, CLS < 0.1, INP < 200ms (mobil først).
  • Canonical korrekt på lister, filtre, paginering og varianter.
  • Parametre: noindex/canonical eller blokering efter strategi.
  • 404/410 ryddet op; 301-kæder minimeret.
  • Hreflang konsistent på alle sprogversioner (inkl. x-default).
  • Structured data valideret (no errors/warnings af relevans).
  • Intern linkstruktur peger autoritet mod de vigtigste sider.
